# Perfect - Offline Installation Setup

This directory contains everything needed to set up Perfect on an offline/air-gapped computer without internet access.

## What's Included

### Node.js (Backend + Frontend)
- **`download_node_packages.sh`** - Downloads all npm packages as tarballs (run on online computer)
- **`setup_node_offline.sh`** - Installs packages on Linux/Mac offline computer
- **`setup_node_offline.bat`** - Installs packages on Windows offline computer
- **`NODE_OFFLINE_SETUP.md`** - Detailed documentation for Node.js offline setup

### Python Client
Located in `../examples/`:
- **`examples/setup_offline.sh`** - Installs Python packages on Linux/Mac offline computer
- **`examples/setup_offline.bat`** - Installs Python packages on Windows offline computer
- **`examples/OFFLINE_SETUP.md`** - Detailed documentation for Python offline setup
- **`examples/wheels/`** - Pre-downloaded Python package wheels

## Quick Start

### Step 1: Prepare on Online Computer

#### For Node.js (Backend + Frontend):
```bash
cd offline_setup
./download_node_packages.sh
```
This creates the `offline_packages/` directory (~200-300 MB) with all npm packages.

#### For Python Client:
The Python wheels are already included in `examples/wheels/` directory. If you need to update them:
```bash
cd examples
python -m pip download -r requirements.txt -d wheels/
```

### Step 2: Transfer to Offline Computer

Copy these directories to your offline computer:
```
Perfect/
├── offline_setup/
│   ├── offline_packages/          # Generated by download_node_packages.sh
│   ├── setup_node_offline.sh
│   ├── setup_node_offline.bat
│   └── NODE_OFFLINE_SETUP.md
├── examples/
│   ├── wheels/                     # Python packages
│   ├── setup_offline.sh
│   ├── setup_offline.bat
│   └── OFFLINE_SETUP.md
├── package.json
├── server/                         # Backend code
├── src/                           # Frontend code
└── ... (all project files)
```

### Step 3: Install on Offline Computer

#### Node.js (Backend + Frontend):

**Linux/Mac:**
```bash
cd offline_setup
./setup_node_offline.sh
```

**Windows:**
```cmd
cd offline_setup
setup_node_offline.bat
```

#### Python Client (Optional):

**Linux/Mac:**
```bash
cd examples
./setup_offline.sh
```

**Windows:**
```cmd
cd examples
setup_offline.bat
```

## Prerequisites

### On Offline Computer:
- **Node.js 18+** (for backend/frontend)
- **Python 3.8+** (for Python client, if needed)
- **~500 MB** free disk space for Node.js
- **~150 MB** free disk space for Python

## Running the Application

After installation:

```bash
# From project root
npm run dev:all      # Starts both frontend and backend
```

Or separately:
```bash
npm run dev          # Frontend only (http://localhost:5173)
npm run server       # Backend only (http://localhost:3000)
```

## Troubleshooting

### Node.js Issues
- See `NODE_OFFLINE_SETUP.md` - Troubleshooting section
- Ensure Node.js 18+ is installed: `node -v`
- Check that `offline_packages/` directory exists

### Python Issues
- See `../examples/OFFLINE_SETUP.md` - Troubleshooting section
- Ensure Python 3.8+ is installed: `python --version`
- Check that `../examples/wheels/` directory exists

### Permission Issues (Linux/Mac)
```bash
chmod +x setup_node_offline.sh
chmod +x ../examples/setup_offline.sh
```

## Updating Dependencies

### Update Node.js packages:
1. Update `../package.json` with new versions (on online computer)
2. Run `./download_node_packages.sh` again
3. Copy new `offline_packages/` to offline computer

### Update Python packages:
1. Update `../examples/requirements.txt` with new versions (on online computer)
2. Run: `cd ../examples && python -m pip download -r requirements.txt -d wheels/`
3. Copy new `wheels/` to offline computer
